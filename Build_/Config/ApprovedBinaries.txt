#
# * Preserve ordering:
#
#   - Preserve sorted order:
#

#     - List binaries within a package in sorted order

#            Changes may impact other codebases so plan build, test and delivery of changes
#            accordingly.
#
#     - No:  It could be that this different version appears because it is an indirect dependency
#            of a newly added package. If so, you can either try to find a compatible version for
#            the newly added package. Alternatively, you can try adding an explicit package
#            reference for the indirect dependency that is of the preferred version. The latter
